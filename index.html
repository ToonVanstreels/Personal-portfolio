Event Clock

The Event Clock is a live schedule tool that allows users to easily see how much time remains until an event starts and how much time is left in an ongoing event. Team members can access the Event Clock on their devices through any web browser. This project was created using HTML, CSS, and JavaScript, and is publicly available via GitHub Pages.

Race Analysis Project

This project creates a post race analysis, using the csv timing results files. The csv files first run trough a Python script to put the data in a uniform order. This way data of different timing providers can be used to create an analysis. A second Pyhton script applies many calculations to the data and creates an HTML report as a final result. A user can open the report in a browser and is able to zoom, hoover and scroll in the report. The above example shows an analysis of the 2025 FIA WEC 6h of Imola Hypercar race.

Event Analysis Project

Unlike the Race Analysis project, the Event Analysis merges all data from an event into a single dataset and generates an interactive HTML report covering the entire event. This allows users to compare results across different sessions and view the progression of performance throughout the event. The above example shows an analysis of a 4 session testday at Spa-Francorchamps.

Brake Simulator

Because the feel of a brake pedal in a race car can differ from that of a race simulator, this simulator plots real-time CAN-bus data of the car’s brake pressure over an ideal brake pressure trace. This allows drivers to train their braking technique in the same car they will drive on track. An Arduino is used to interpret the CAN messages, while a Python script generates and plots the brake traces. It is also possible to overlay an onboard video for additional reference.
